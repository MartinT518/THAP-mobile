# Repository Comparison: New PWA vs Old Flutter App

This document provides a comprehensive comparison between the **current thap-mobile repository** (React/TypeScript PWA) and the **old thap_mobile_clone repository** (Flutter native app). Each section uses a two-column table with the new repo in the first column and the old repo in the second.

---

## 4. Summary of Key Differences

### What the new PWA added (not in old app)

- Direct Google Sign-In via Google Identity Services (replacing Manus OAuth portal)
- AI chat with BYOK multi-provider support (OpenAI, Gemini, Perplexity, DeepSeek)
- AI settings and API key management
- Full backend in the same repository (Express + tRPC + Drizzle + MySQL)
- Generalized document management system (S3-backed, with receipt/photo/manual/note types)
- Comprehensive server-side test suite (20+ test files)
- PWA with service worker, offline support, and installability
- Icecat product data integration
- Server-side OpenGraph scraping (no third-party API dependency)
- Notification settings and privacy settings pages
- Admin utilities (cleanup, demo seed, system health)
- Analytics integration (Umami)
- Duplicate product cleanup
- Chinese (`zh`) and Norwegian (`no`) language support
- Web Share API integration with copy-link fallback
- QR code generation for owned products (`qrcode.react`) with download-as-image
- Link-based product sharing with accept/dismiss flow (`product_shares` table + `sharing.*` tRPC procedures)
- "Shared with me" section on Home page
- Tag chip filtering on inventory views
- List/grid view toggle for product inventory
- Camera capture hint for photo/receipt uploads on mobile (camera-first + gallery option)
- External link "return to app" toast affordance (PWA substitute for in-app WebView)

### What the old Flutter app had (not in new PWA)

- Native mobile app experience (App Store / Play Store distribution)
- Buy buttons and action shortcuts in product detail pages
- In-app browser / WebView for external links (PWA now has "return to app" toast pattern as substitute)
- Lithuanian (`lt`), Latvian (`lv`), and Dutch (`nl`) language support
